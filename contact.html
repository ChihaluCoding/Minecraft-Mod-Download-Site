バグ報告のお願い

下記のポイントを参考に Issue を作成していただくと迅速に確認できます。

共有してほしい情報

Issues には以下の情報を箇条書きで記載してください。情報量が多いほど、原因調査と修正の優先判断がスムーズになります。

記入例

  • Mod名
  • 発生した症状と期待する挙動
  • 再現手順（スクリーンショット / ログがあればリンクでも可）
  • 使用している Minecraft / ローダー / Mod のバージョン
  • 導入している関連 Mod やリソースパック

Issues 作成前のチェック

  • 他のModを無効化して再現するか確認した
  • 既存の Issues に同じ内容がないか検索した
  • 最新バージョンでバグが生じるか確認した
  • ログ・スクリーンショットを用意した
  • 問題の発生条件を整理した

準備が整っているほど、修正が円滑に進みます。

1

Issues を新規作成

リポジトリの「New issue」ボタンからバグ報告用の Issues を作成します。

2

情報を入力

上記の共有内容を参考に、再現手順や環境情報を箇条書きで記載してください。

3

フォローアップ

追加情報が分かった場合は、同じ Issues にコメントで追記をお願いします。

GitHub Issues を開く
報告したバグが修正されたかどうかはどうやってわかりますか?

Issues のステータスを確認するか、修正がリリースされた際にお知らせします。

修正が却下されることはありますか?

はい、あります。

具体的には、以下のようなケースです。

  • 他のModとの互換性の問題
  • ゲームの仕様に依存するバグ
  • 修正が他の機能に影響を与える場合
旧バージョンのModのサポートはありますか?

基本的には最新バージョンのサポートに注力していますが、旧バージョンの問題についても対応する場合があります。

最新バージョンの修正が優先しているため、旧バージョンの修正は後回しになることがあります。

ただし、アーカイブ済みというタグが付けられたModについては、基本は対応しません。

GitHub Issues 以外の報告方法はありますか?

基本的には GitHub Issues を通じての報告をお願いしていますが、報告方法が分からない場合のみ、X（旧Twitter）@ChihaluCodingでの問い合わせも受け付けています。
